# Plan: Gemini CLI harness adapter (#68)

## Goal

Add **Gemini CLI** (Google, v0.35.0) as the third qualifying harness behind the
mature Phase-0/`#67` contract (`src/harness/`). After this, `abs install-hooks
--harness gemini` + `abs setup --harness gemini` wire Gemini's real hooks +
MCP server, Gemini sessions are captured/recalled/guarded through the same
canonical lifecycle, and Gemini observations are namespaced `gemini:` so they
never collide with Claude (bare) or Codex (`codex:`) session ids on one DB.

The cross-cutting work is **already solved** by `#67`: the dispatch chokepoint
namespaces `payload.sessionId` exactly once (`src/hooks/dispatch.ts:60`), the
parser-dispatch seam exists in `ingestFile` (`src/ingest/ingest.ts:271`), the
hook payload parser already reads `session_id`/`transcript_path`
(`src/hooks/payload.ts:54`), and the canonical handler args (`session-end`,
`session-start`, `user-prompt-submit`, `pre-tool-use`) are harness-agnostic. So
this issue is five small, mostly-additive pieces ÔÇö strictly lower risk than #67.

## Architecture

The harness contract (`src/harness/types.ts`) is unchanged. We compose existing
capabilities where the on-disk reality matches, and add **two narrow variants**
where it does not:

1. **Namespacing** (leaf, `src/ingest/namespacing.ts`): add `isGeminiTranscript`
   + a third branch in `harnessForPayload`. Migration-safe: Claude bare, Codex
   `codex:`, Gemini `gemini:`. **Reuses** the existing chokepoint verbatim.
2. **Lifecycle installer**: Gemini hooks live in `~/.gemini/settings.json` under
   `hooks.<Event>[]` matcher groups ÔÇö **byte-identical JSON shape** to Claude's
   settings.json. But the *event names differ* (`SessionEnd`/`SessionStart`/
   `BeforeAgent`/`BeforeTool`, not `UserPromptSubmit`/`PreToolUse`), and the
   `HOOK_REGISTRY` in `src/hooks/installer.ts` is hardcoded to Claude event
   names, so `installHooks` *cannot* emit Gemini keys. Ôćĺ **Gemini variant**
   modeled on the Claude installer's safety machinery, with a Gemini hook table.
3. **MCP registrar**: Gemini settings.json `mcpServers` map has the same shape
   (`{command, args, env}`), and `gemini mcp add` exists ÔÇö BUT the real binary
   **rejects the `--` separator** (verified on disk; it prints help). It needs
   the *positional* form `gemini mcp add <name> --scope user node <cli> start`.
   Ôćĺ extend `setup.ts` with a minimal arg-style option; **reuses** the
   probe/list/add control flow.
4. **Transcript source**: Gemini transcripts are a **single JSON file** (NOT
   JSONL) ÔÇö `~/.gemini/tmp/<project>/chats/session-*.json` holding one
   `ConversationRecord` object that Gemini **rewrites whole on every message**
   (`chatRecordingService.js:360`, full overwrite). `jsonlTranscriptSource`/
   `ingestSingleSession` stream line-by-line and cannot parse it. Ôćĺ **Gemini
   variant**: a `gemini-chat.ts` parser + a routing branch in `ingestFile` that
   reads the whole file as JSON **and gates re-ingest by a persistent
   per-session ID-ANCHORED watermark** (last-ingested message `id`, NOT byte
   size and NOT a raw count ÔÇö see C1 below). A byte cursor is meaningless for a
   whole-file rewrite: a resumed session re-fires `SessionEnd` on the grown file,
   `cursor < size` re-reads from 0, and because `createObservation`
   (`memory-store.ts:326`) is a **plain INSERT with no unique constraint and no
   `ON CONFLICT`**, every prior message is re-stored ÔÇö a duplicate explosion. The
   id-watermark skips the already-ingested prefix and **re-syncs (not drops) on
   `/rewind`** (W-NEW-1). The cwd is NOT in the JSON, so the branch reads it from
   the `.project_root` marker so Gemini sessions bucket under the REAL project,
   never the literal `"chats"` (C-NEW-1).
5. **Adapter** (`src/harness/adapters/gemini.ts`): composes the above + detect
   (`~/.gemini`), `qualifies` (all four pillars present), `eventMap`,
   `resolveSession` (payload-first; Gemini has no session-id env var).
6. **Register** in `defaultRegistry()`.

## Ground truth (Gemini, verified on disk ÔÇö v0.35.0 at `/opt/homebrew/bin/gemini`)

All findings below were read off the real install / generated by running the real
binary in a throwaway `$HOME`, NOT from research.

### Config + settings location

- Config root: `~/.gemini/`. **There is no `~/.gemini/settings.json` until something
  writes one.** `~/.gemini/config/mcp_config.json` exists but is **empty (0 bytes)**
  and is NOT where `mcp add` writes.
- `gemini mcp add --scope user ÔÇŽ` writes **`~/.gemini/settings.json`**;
  `--scope project` writes **`<cwd>/.gemini/settings.json`**. Scope default is
  `project`. Verified written shape:

  ```json
  {
    "mcpServers": {
      "abs2": { "command": "node", "args": ["/cli.js", "start"] }
    }
  }
  ```

  With `-e FOO=bar` it adds `"env": { "FOO": "bar" }`. So the `mcpServers` value
  is structurally identical to Claude's settings.json `mcpServers` entry.

### MCP register command (verified ÔÇö the `--` separator is REJECTED)

```
gemini mcp add <name> <commandOrUrl> [args...]
  --scope user|project (default project)
  -t/--transport stdio|sse|http (default stdio)
  -e KEY=value, -H "Header: v", --trust, --timeout, --description
gemini mcp list      # "ÔťŚ name: node /cli.js start (stdio) - Disconnected"
gemini mcp remove <name>
gemini mcp enable|disable <name>
```

- **WORKS** (positional): `gemini mcp add agentbrainsystem --scope user node <cli> start`
  Ôćĺ `MCP server "agentbrainsystem" added to user settings. (stdio)`.
- **FAILS** (Codex/Claude `--` style): `gemini mcp add agentbrainsystem --scope user -- node <cli> start`
  Ôćĺ prints the help text (yargs parse failure), writes nothing. So the existing
  `buildMcpAddArgs` (`['mcp','add',NAME,'--','node',cli,'start']`) is incompatible.
- `gemini mcp add` **fails fast on missing auth** unless `GEMINI_API_KEY` (or
  `GOOGLE_GENAI_USE_VERTEXAI`/`GOOGLE_GENAI_USE_GCA`) is set ÔÇö it exits non-zero
  BEFORE writing. The PRODUCT path handles this cleanly with no new code: the
  non-zero exit becomes `status:'error'` Ôćĺ `cmdSetup` prints the (now positional,
  via W2) manual command non-fatally. **No dummy key in product code.** A dummy
  key happens to satisfy the local-write gate, but that is a TEST-ONLY
  convenience for generating fixtures under a throwaway `$HOME` ÔÇö the product
  must NOT inject one (see W3 in Task 5).

### Hooks ÔÇö event names + config shape + payload (read from gemini-cli-core dist)

`gemini hooks` only exposes one subcommand: `gemini hooks migrate [--from-claude]`
(migrate Claude Code hooks Ôćĺ Gemini). There is **no `hooks add`** ÔÇö hooks are
declared directly in settings.json under a top-level `hooks` key, merged into
config (`config.js:591 this.hooks = params.hooks`).

Real event enum (`hooks/types.d.ts` `HookEventName`):

```
BeforeTool, AfterTool, BeforeAgent, Notification, AfterAgent,
SessionStart, SessionEnd, PreCompress, BeforeModel, AfterModel,
BeforeToolSelection
```

Hook config shape (`HookDefinition`) ÔÇö the **same** `matcher` + `hooks[]` shape
as Claude, with `{type:"command", command, timeout?}` entries:

```ts
interface HookDefinition { matcher?: string; sequential?: boolean; hooks: HookConfig[]; }
interface CommandHookConfig { type: "command"; command: string; timeout?: number; env?; }
```

So settings.json would hold, e.g.:

```json
{
  "hooks": {
    "SessionEnd":  [{ "matcher": "", "hooks": [{ "type": "command", "command": "abs hook session-end", "timeout": 30 }] }],
    "SessionStart":[{ "matcher": "", "hooks": [{ "type": "command", "command": "abs hook session-start", "timeout": 10 }] }],
    "BeforeAgent": [{ "matcher": "", "hooks": [{ "type": "command", "command": "abs hook user-prompt-submit", "timeout": 10 }] }],
    "BeforeTool":  [{ "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "abs hook pre-tool-use", "timeout": 5 }] }]
  }
}
```

Hook **payload is Claude-compatible** (`HookInput`, `hooks/types.d.ts:98`):

```ts
interface HookInput {
  session_id: string;       // full UUID
  transcript_path: string;  // ChatRecordingService.getConversationFilePath()
  cwd: string;
  hook_event_name: string;
  timestamp: string;        // ISO
}
```

`parseHookPayload` (`src/hooks/payload.ts:54`) already reads `session_id` +
`transcript_path`, so **no payload-parser change is needed**.

Event Ôćĺ moment mapping decision:

| Gemini event  | Canonical moment | `abs hook` arg        | Why |
|---------------|------------------|-----------------------|-----|
| `SessionEnd`  | capture          | `session-end`         | Gemini HAS SessionEnd (unlike Codex). `SessionEndInput` carries `reason`. |
| `SessionStart`| recall           | `session-start`       | `SessionStartInput.source` = startup/resume/clear. |
| `BeforeAgent` | recall           | `user-prompt-submit`  | `BeforeAgentInput.prompt` ÔÇö the per-turn recall trigger (UserPromptSubmit analog). |
| `BeforeTool`  | guard            | `pre-tool-use`        | `BeforeToolInput.tool_name` + `tool_input` ÔÇö the contradiction guard. Scope matcher to `Edit|Write` like Claude. |

`AfterTool`/`AfterAgent`/`Notification`/`PreCompress`/`BeforeModel` are unused.

### Transcript ÔÇö single JSON file (NOT JSONL), real sample captured

`transcript_path` = `ChatRecordingService.getConversationFilePath()`
(`hookEventHandler.js:228`). The recorder
(`services/chatRecordingService.js`) writes ONE JSON file per session:

- Dir: `<getProjectTempDir()>/chats/` = `~/.gemini/tmp/<project>/chats/`.
- Filename: `session-<YYYY-MM-DDTHH-MM>-<sessionId[:8]>.json`
  (const `SESSION_FILE_PREFIX = "session-"`).
- **`<project>` is a SLUG of the cwd basename**, NOT a hash and NOT the full cwd ÔÇö
  verified: the real `~/.gemini/tmp/agentbrainsystem/` exists. (The d.ts comment
  says `<project_hash>`; the running v0.35.0 uses a short slug. The parser must not
  assume a hash.) `getProjectTempDir()` =
  `<globalTmp>/<projectIdentifier>`, and `projectIdentifier = registry.getShortId(cwd)`
  (`config/storage.js:176`). `getShortId` Ôćĺ `claimNewSlug` Ôćĺ `slugify(basename(cwd))`
  with a `-N` collision suffix (`config/projectRegistry.js:186-192`). So the dir
  name is `slugify(basename(cwd))`, e.g. cwd `/Users/x/Devs/agentbrainsystem` Ôćĺ
  slug `agentbrainsystem`. It is a slug of the LAST path segment, not the cwd.

- **The REAL cwd is recoverable on disk ÔÇö TWO sources, verified (C-NEW-1).** Each
  slug dir carries a marker file `<slugDir>/.project_root` whose contents are the
  **absolute normalized cwd** (`config/projectRegistry.js:235-247`, written with
  `flag:'wx'`). Verified on the live install:
  `cat ~/.gemini/tmp/agentbrainsystem/.project_root` Ôćĺ
  `/Users/dev/Devs/agentbrainsystem`. There is also a registry
  `~/.gemini/projects.json` = `{ "projects": { "<cwd>": "<slug>" } }` (inverse map).
  From a chat file `ÔÇŽ/tmp/<slug>/chats/session-*.json`, the marker is exactly
  `join(dirname(dirname(absPath)), '.project_root')` ÔÇö one dir up from `chats/`.

Real file generated by `gemini -p "say hi"` (full content):

```json
{
  "sessionId": "78432a44-385f-41f6-8a71-646d51996f8a",
  "projectHash": "0d1a4009f6700896c0146f7f6a9cff700c8b147e430d24c26143836167a239cc",
  "startTime": "2026-05-23T04:24:24.551Z",
  "lastUpdated": "2026-05-23T04:24:24.551Z",
  "messages": [
    {
      "id": "93a3561e-ace3-46b8-82e8-d24cecd6c0e9",
      "timestamp": "2026-05-23T04:24:24.551Z",
      "type": "user",
      "content": [ { "text": "say hi" } ]
    }

  "kind": "main"

```

Per-message schema (`ConversationRecord` / `MessageRecord` in
`chatRecordingService.d.ts`):

```ts
interface ConversationRecord {
  sessionId: string; projectHash: string; startTime: string; lastUpdated: string;
  messages: MessageRecord[]; summary?: string; directories?: string[];
  kind?: "main" | "subagent";
}
// VERIFIED (chatRecordingService.d.ts:68-78, real file generated on disk):
// There is NO cwd field. `projectHash` is sha256(cwd) ÔÇö NOT reversible to a path.
// `directories?` is "workspace directories added during the session via /dir add"
// (d.ts:75 comment) ÔÇö it is EMPTY/ABSENT for a normal session and is NOT the cwd.
// Ôćĺ the cwd is NOT in the JSON; it must come from the `.project_root` marker (above).
type MessageRecord = { id: string; timestamp: string; content: PartListUnion;
  displayContent?: PartListUnion } & (
    { type: "user" | "info" | "error" | "warning" }
  | { type: "gemini"; toolCalls?: ToolCallRecord[]; thoughts?; tokens?; model? } );
```

- `content` is `PartListUnion` = `string` OR `Part[]` (each `Part` is `{text}`
  or `{functionCall}`/`{functionResponse}` from `@google/genai`). User text above
  is `[{ "text": "say hi" }]`.
- Role mapping: `type:"user"` Ôćĺ user prose; `type:"gemini"` Ôćĺ assistant prose;
  `type:"info"|"error"|"warning"` Ôćĺ skip (UI chrome). `toolCalls` on gemini
  messages are the anchor source (prose-only for MVP, like the Codex parser).
- **Session id is in the file** (`sessionId`, full UUID) AND derivable from the
  filename `[:8]` slice + payload `session_id`. The payload's `session_id` is the
  authoritative full UUID; the parser will also read the in-file `sessionId`.
- **Each message carries a stable `id` (`randomUUID()`, `chatRecordingService.js:113`)**
  ÔÇö load-bearing for the rewind watermark (W-NEW-1). Verified in the generated
  file: `messages[0].id = "6f708c3e-ÔÇŽ"`.
- **`/rewind` TRUNCATES the array (W-NEW-1).** `rewindTo(messageId)` does
  `conversation.messages = messages.slice(0, messageIndex)` then rewrites the whole
  file (`chatRecordingService.js:560-568`). A pure count-watermark therefore breaks:
  count N, rewind NÔćĺK, then regrow KÔćĺM; if M ÔëĄ N the `entries.length > already`
  guard is false Ôćĺ the new (post-rewind) turns are SILENTLY DROPPED. Fix is an
  id-anchored watermark (Task 4 C1).

```ts
/** True when the path is a Gemini CLI chat transcript (drives parser + namespace). */
export function isGeminiTranscript(absPath: string): boolean {

    /\/chats\/session-[\dT-]+-[0-9a-f]{8}\.json$/i.test(absPath)
```

```ts
  // ENTIRE file on every message (append-only EXCEPT `/rewind`, which truncates).
  // A byte cursor is meaningless; re-ingest is gated by a persistent per-session
  // ID-ANCHORED watermark (W-NEW-1): we remember the LAST-ingested message `id`
  // and ingest everything AFTER it. If that id is gone (rewound past it), we
  // RE-SYNC from the start ÔÇö at-least-once dup, but NEVER a silent drop.
  const raw = await readFile(absPath, 'utf8');               // node:fs/promises

  // `.project_root` marker one dir up from `chats/`. The marker IS the cwd
  // (matches Claude/Codex projectSlug(cwd)). Fallback: the slug dir name. NEVER 'chats'.

    : basename(dirname(dirname(absPath)));                    // safety net = the <slug> dir, never 'chats'

    start = idx >= 0 ? idx + 1 : 0;                          // found Ôćĺ tail; NOT found Ôćĺ re-sync (rewound)
  }
  for (let i = start; i < entries.length; i++) {
    const entry = entries[i];

    const entryProject = entry.cwd ? projectSlug(entry.cwd) : effectiveProject;
    await writeEntry(memory, sessionCache, bindingCache, externalId, entryProject, entry, absPath, tally);
  }
  // Advance the watermark to the LAST entry's id (the new tail end). On an
  // unchanged file `start === entries.length` Ôćĺ nothing written, id unchanged.
  const tailId = entries.at(-1)?.id;
  if (tailId) memory.store.setMeta(lastIdKey, tailId);
  result.observationsAdded += tally.added;
  result.observationsSkipped += tally.skipped;
```

**C1 ÔÇö persistent per-session ID-ANCHORED watermark (dedup + rewind-safe).**
Gemini rewrites the whole JSON on every message, so a byte cursor
(`writeCursor(size)`) is the wrong granularity: a resumed session re-fires
`SessionEnd` on the grown file, `cursor < size` triggers a re-read from 0, and
`createObservation` (`memory-store.ts:326`) is a **plain INSERT ÔÇö no unique
constraint, no `ON CONFLICT`** ÔÇö so every prior message is re-stored. The `seen`
set inside any parser is **per-ingest-run** (fresh each call), so it gives no
cross-run dedup (the Codex parser documents exactly this at
`codex-jsonl.ts:96-99`). There is **no** `(session, normalized-text)`
idempotency anywhere; the earlier claim was false.

A pure **count** watermark N fixes the steady-state dup explosion BUT breaks on
`/rewind` (W-NEW-1): `rewindTo` does `messages.slice(0, idx)` then rewrites
(`chatRecordingService.js:560-568`) ÔÇö a genuine truncation. After rewind NÔćĺK then
regrow KÔćĺM, if M ÔëĄ N the `entries.length > already` guard is false Ôćĺ the new
post-rewind turns are **silently dropped** (worst-case data loss). So the
watermark must key off message **identity**, not count.

Fix ÔÇö **id-anchored watermark.** Each message carries a stable `id`
(`randomUUID()`, `chatRecordingService.js:113`). Store the **last-ingested message
`id`** per session-file:

- **kv_meta key**: `GEMINI_LASTID_PREFIX + absPath` where
  `GEMINI_LASTID_PREFIX = 'gemini:lastid:'` (keyed by `absPath`, parallel to the
  existing `CURSOR_PREFIX`/`CODEX_CWD_PREFIX` convention; the file path is 1:1
  with the session, survives across runs in `kv_meta`).
- **find-tail**: on re-ingest, locate that id's index in the CURRENT parsed
  `entries`; ingest everything AFTER it (`start = idx + 1`).
- **re-sync on rewind**: if the stored id is NOT found (rewound past it), set
  `start = 0` and re-ingest the whole file. This yields at-least-once duplicates
  ÔÇö ACCEPTABLE (same tolerance class as the existing at-least-once design and
  Codex W2/W4) ÔÇö but NEVER silently drops new turns.
- **advance**: after the loop, set the key to the LAST entry's `id`
  (`entries.at(-1)?.id`). On an unchanged file the stored id is the tail id Ôćĺ
  `start === entries.length` Ôćĺ 0 new observations (idempotent).
- The watermark indexes the **parsed** (`user`/`gemini` prose) entry list, which
  is what `parseGeminiChat` returns and what we skip over ÔÇö info/error/warning
  chrome is dropped before the list, so the ids we anchor on are always present
  in a re-parse of the same (or grown) file.

No new read/write helpers are needed beyond the store's existing
`getMeta`/`setMeta` (used inline in the branch above, mirroring the Codex
`CODEX_CWD_PREFIX` `getMeta`/`setMeta` pattern). Declare only the two prefixes
next to `CURSOR_PREFIX`/`CODEX_CWD_PREFIX` in `ingest.ts`:

```ts

 * The real cwd for a Gemini chat file (C-NEW-1). The chat JSON has no cwd; Gemini
 * stores it in `<ÔÇŽ/tmp/<slug>>/.project_root`, one dir up from `ÔÇŽ/chats/<file>`.

function readGeminiProjectRoot(absPath: string): string | undefined {
```

```ts
import type { ParsedEntry } from './claude-jsonl.js';

export function parseGeminiChat(raw: string, absPath: string): ParsedEntry[] {
  let doc: unknown;
  try { doc = JSON.parse(raw); } catch { return []; }   // malformed Ôćĺ skip whole file
  if (!isRecord(doc) || !Array.isArray((doc as any).messages)) return [];
  const sessionId = asString((doc as any).sessionId) ?? sessionIdFromFilename(absPath);
  if (!sessionId) return [];
  const out: ParsedEntry[] = [];
  for (const m of (doc as any).messages as unknown[]) {
    if (!isRecord(m)) continue;
    const type = asString(m.type);
    if (type !== 'user' && type !== 'gemini') continue;     // skip info/error/warning chrome
    const id = asString(m.id);
    if (!id) continue;                                      // no id Ôćĺ cannot watermark; skip (every real msg has one)
    const role = type === 'gemini' ? 'assistant' : 'user';
    const text = clean(extractParts(m.content));            // flatten PartListUnion Ôćĺ prose
    if (!text) continue;                                    // prose-only MVP (anchors are follow-on)
    out.push({ sessionId, role, text, toolAnchors: [], id });
  }
  return out;
}
```

- `extractParts(content)`: `string` Ôćĺ as-is; `Part[]` Ôćĺ join `p.text` of each
  `{text}` part; ignore `{functionCall}`/`{functionResponse}` parts (prose only).
- `clean`: reuse the injected-wrapper / blank-collapse helper pattern from the
  Claude/Codex parsers (`<system-reminder>` etc. stripping) ÔÇö extract a shared
  `clean` if trivial, else mirror it.
- `sessionIdFromFilename`: `session-<ts>-<8hex>.json` Ôćĺ only the 8-hex slice is
  in the name, so prefer the in-file `sessionId` (full UUID); the filename slice
  is a last-resort and would mis-namespace vs the payload's full UUID ÔÇö so if the
  in-file `sessionId` is absent, return `undefined` (skip) rather than guess a
  truncated id. (The payload always carries the full `session_id`; the parser path
  is the SessionEnd auto-ingest where the file's `sessionId` is always present.)

```ts
// src/harness/adapters/gemini.ts
import { access } from 'node:fs/promises';
import { homedir } from 'node:os';
import { join } from 'node:path';
import { geminiLifecycleInstaller } from '../capabilities/gemini-lifecycle-installer.js';
import { cliMcpRegistrar } from '../capabilities/mcp-registrar.js';
import { payloadFirstResolver } from '../capabilities/session-resolver.js';
import type { HarnessAdapter } from '../types.js';

/**
 * Gemini CLI adapter (#68) ÔÇö third qualifying harness. JSON settings.json hooks
 * (Gemini event names, Task 3 installer), positional `gemini mcp add` (no `--`,
 * Task 5), single-JSON transcript (Task 4). PAYLOAD-ONLY session resolver: Gemini
 * exposes no session-id env var, so no Claude-style env leaks in. Gemini HAS
 * SessionEnd (capture), unlike Codex.
 */
export function geminiAdapter(): HarnessAdapter {
  const resolve = payloadFirstResolver(); // payload-only ÔÇö no GEMINI_* session-id env
  const installer = geminiLifecycleInstaller();
  const registrar = cliMcpRegistrar({ binary: 'gemini', argStyle: 'positional', scope: 'user' });
  return {
    id: 'gemini',
    displayName: 'Gemini CLI',
    mcpBinary: 'gemini',
    detect: async () => { try { await access(join(homedir(), '.gemini')); return true; } catch { return false; } },
    qualifies: () => ({ ok: true, missing: [] }),
    eventMap: {
      capture: ['SessionEnd'],
      recall: ['SessionStart', 'BeforeAgent'],
      guard: ['BeforeTool'],
    },
    install: () => installer.install(),
    uninstall: () => installer.uninstall(),
    registerMcp: (cliPath, run) => registrar.register(cliPath, run),
    resolveSession: (input) => resolve(input),
  };
}
```
